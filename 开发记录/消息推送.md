# iOS JPush 证书更新完整步骤（最终版）

## 一、在 Apple Developer 生成推送证书

1. 登录 Apple Developer → Certificates
    
2. 对每个 App 分别生成 **2 种证书**：
    
    - **Development / Sandbox** 推送证书
        
    - **Production** 推送证书
        
3. 生成时会用同一个 CSR（可以共用 CSR）
    
4. 下载每一个产出的 `.cer` 文件
    

---

## 二、导入到钥匙串 & 导出 .p12

1. 双击每一个 `.cer`，自动导入钥匙串
    
2. 打开「钥匙串访问」 → 登录 → 证书
    
3. 找到对应证书：
    

|证书名字|类型|
|---|---|
|`Apple Push Services: com.xxx.xxx`|Production 生产证书|
|`Apple Development iOS Push Services: com.xxx.xxx`|Development / Sandbox 开发证书|

4. 一定要选中 **证书 + 私钥一起** 右键导出 → `.p12`
    
5. 导出时设置一个密码（记住！极光上传要用）
    

---

## 三、上传到极光

1. 登录极光后台 → 应用 → 设置 → iOS证书文件
    
2. Production p12 → 上传到「生产环境」区域
    
3. Development p12 → 上传到「开发环境」区域
    
4. 输入 .p12 密码 → 保存
    
5. 上传后确认极光显示新的有效期
    

---

## 四、清理旧证书（建议）

当新证书已经成功导出p12并上传极光后  
可以在钥匙串把旧的过期证书删除  
避免以后导 p12 时搞混。

---

## 五、注意事项

- 每个 App 都需要各自一套推送证书（生产+开发）
    
- 生成时 CSR 可以共用
    
- 换证书 **不需要重新上架 App**
    
- Apple push 证书有效期一般 1 年，建议提前 7~10 天更换
    
- 名称区分永远看钥匙串内 「Development」和「Push Services」字样
